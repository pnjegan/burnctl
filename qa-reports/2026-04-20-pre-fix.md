## === FAIL (9) ===

### [CRITICAL] BUG-1 work-timeline — maintainer-path leak
- **Expected:** "No burnctl database found. Run `burnctl scan` from your project dir first."
- **Actual:** Shows real timeline data (Claudash $227.99, Apr 20 SURFACE PATTERN, CC active time) from fresh `/tmp/` install with no local DB.
- **Evidence:**
  ```
  Apr 20 Work Intelligence
  SURFACE PATTERN
  ```
- **Hypothesis:** `work_timeline.py:41-46` has hardcoded fallback `os.path.expanduser("~/projects/burnctl/data/usage.db")`.
- **Fix reference:** Copy `overhead_audit.py::load_db()` — drops only to `~/.burnctl/` fallback, never `~/projects/burnctl/`.

### [HIGH] BUG-2 /api/stats — HTTP 404
- **Expected:** JSON response with stats payload.
- **Actual:** `HTTP/1.0 404 Not Found` + auto-refresh error HTML.
- **Evidence:** `curl -si http://localhost:8080/api/stats` → `HTTP/1.0 404 Not Found`.
- **Hypothesis:** Dashboard template (`templates/dashboard.html`) fetches `/api/stats` but route not registered in `server.py`.
- **Fix:** Either add route or rename template fetch to existing endpoint.

### [HIGH] BUG-3 resume-audit — false positives (63% flag rate)
- **Expected:** Signal-to-noise ratio that users can act on.
- **Actual:** Flags 112 of 178 sessions (63%) including sessions with 91-95% cache hit rate.
- **Evidence:**
  ```
  🔴 [2026-03-23 14:47] subagents
     Input: 4,831,600 tokens | Cache hit: 94.7% | 5m TTL: 100.0%
  ```
  A 94.7% cache hit rate is healthy — flagging is noise.
- **Hypothesis:** Criterion is `5m_TTL_dominant OR low_cache_hit` (OR instead of AND).
- **Fix:** Change to `5m_TTL_dominant AND cache_hit < 50%`.

### [MEDIUM] BUG-4 duplicate fixes in DB
- **Expected:** Each `(project, fix_type)` pair unique or warn on duplicate insert.
- **Actual:** 3 dupes on `claude_md`, 3 dupes on `claude_md_rule`.
- **Evidence:**
  ```sql
  SELECT fix_type, COUNT(*) FROM fixes GROUP BY fix_type HAVING COUNT(*)>1
  → [('claude_md', 3), ('claude_md_rule', 3)]
  ```
- **Fix:** Dedup check on write in `fix_apply.py`.

### [MEDIUM] BUG-5 / BUG-6 "Claudash" project name leaks publicly
- **Expected:** Post-rebrand, no "Claudash" in user-visible API responses.
- **Actual:** `/api/projects` returns `"name": "Claudash"` — top-2 by token share (46.9%), $2,743 cost.
- **Evidence:** `curl /api/projects` → project names include `Claudash`.
- **Hypothesis:** Scanner derives project name from directory (`~/projects/Claudash/`). Legacy rename never backfilled.
- **Fix:** Name remap in `server.py` or `scanner.py`: `Claudash → burnctl`.

### [LOW] BUG-7 version-check — misleading interceptor warning

  ✓ Not in known-bad range (2.1.69 – 2.1.89)
  ℹ  Cache-fix interceptor NOT installed     ← misleading

- **Fix:** Only surface interceptor message if user IS on bad version range.
